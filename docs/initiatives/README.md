# Initiatives Index

`docs/initiatives/` is the initiative-output side of the repo.

## What Lives Here

- active product docs
- active architecture and rollout docs
- review and completion records
- initiative-local archives
- repo-level governance surfaces for initiative currentness:
  - `NOW.md`
  - `DOC_INDEX.md`
  - `TIMELINE.md`

## What Does Not Live Here

- role prompts
- startup bootstraps
- cross-team governance
- operations runbooks

Those belong under `docs/workflow/`.

## Placement Standard

- If a file defines current initiative truth, place it under the initiative itself.
- If a file records initiative history, archive it under that same initiative.
- Keep output docs and historical archive together so retrieval follows the business line, not the team org chart.
- Keep repo-level currentness surfaces at the root of `docs/initiatives/`, not buried inside one initiative.
