# Workflow Docs

`docs/workflow/` is the people-and-process side of the repo.

## What Lives Here

- governance
  - decision rules, document-state rules, branch discipline
- operations
  - runbooks, tool instructions, deployment coordination
- teams
  - role-owned prompts, handoffs, team README files, dispatch scaffolds
- startup-prompts
  - copy-ready first-message bootstraps for brand-new owner and worker dialogs

## What Does Not Live Here

- PRDs
- architecture baselines
- rollout truth
- initiative reviews or completion records
- historical archive snapshots

Those belong under `docs/initiatives/`.

## Placement Standard

- If the file answers "how should this role work", put it in `docs/workflow/`.
- If the file answers "what is true for this initiative right now", put it in `docs/initiatives/`.
- If the file exists only to bootstrap a fresh chat, put it in `docs/workflow/startup-prompts/`.
