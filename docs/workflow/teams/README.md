# Team Docs

This folder is organized by functional line, so prompts and handoffs stay attached to the people who own them.
It is part of `docs/workflow/`, not `docs/initiatives/`, because these files define how teams work rather than what an initiative currently is.

## Current Lines

- `shared/`
  - cross-team startup or onboarding prompts
- `engineering/`
  - architecture owner and worker execution docs
- `product/`
  - product owner prompts, reviews, and assignments
- `experience/`
  - UI and interaction design prompts and review notes
- `user-insight-copy/`
  - user psychology, persuasion, and copy direction

## Placement Rule

- Team-level prompts belong here even if they are Markdown prompt files.
- Do not put role-owned prompts back into a generic `docs/prompts/` directory.
- Do not store PRDs, rollout truth, review records, or archive snapshots here; those belong under `docs/initiatives/`.
